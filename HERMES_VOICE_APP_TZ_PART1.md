# Hermes Voice App — Полное техническое задание

## 1. Введение

**Цель:** Создать Android-приложение для голосового управления через Telegram.

**Архитектура:** Двухагентная система:
- **Hermes Agent** (сервер, 91.132.57.16) — ИИ-ассистент на базе DeepSeek, STT (whisper), TTS (Piper)
- **Android App** (Hermes Voice) — голосовой интерфейс на телефоне пользователя

**Связь:** Через Telegram Bot API (@hermvois_bot). Приложение слушает микрофон, отправляет голосовые сообщения в бот, получает голосовые ответы и воспроизводит их.

**Язык:** Весь интерфейс и взаимодействие на **русском языке**.

## 2. Текущее состояние (v2.1.0)

### 2.1 Что работает
- Кнопка Вкл/Выкл — запускает/останавливает микрофон
- Push-to-Talk (PTT) — запись только при удержании кнопки
- Отправка WAV/OGG в Telegram Bot API через @hermvois_bot

## 3. Что нужно сделать (задачи для LLM)

### Задача 1: Wake Word Detection

**Требование:** Приложение должно постоянно слушать микрофон. Когда пользователь произносит кодовое слово «Гермес» — начинать запись команды. Когда тишина 1.5 сек — отправлять запись в Telegram.

**Критерии:**
- Полностью **офлайн**, без интернета
- Низкое энергопотребление (работает часами в фоне)
- Низкая задержка (<500ms от слова до начала записи)
- Минимум false positives
- Русский язык поддержка

**Найденный кандидат:** Porcupine от Picovoice (`porcupine_flutter: ^4.0.0`)
- Плюсы: 2MB размер, офлайн, русский поддерживает, есть Flutter SDK
- Минусы: нужен AccessKey (бесплатный), кастомное слово через Picovoice Console

**Что искать:**

- Можно ли использовать Android Voice Interaction API?
- Есть ли готовые Flutter плагины, кроме Porcupine?
- Какие есть альтернативы с открытым исходным кодом (Vosk, Kaldi, Snowboy)?
- Что лучше всего подходит для русского wake word «Гермес»?

### Задача 2: Always-On микрофон (прослушивание)

**Требование:** После нажатия «Включить» — приложение должно работать в фоне, слушать микрофон, быть устойчивым к убиванию Android-системой.

**Текущая реализация:**
- Foreground Service с уведомлением
- AudioRecord напрямую (без MediaRecorder)

**Что проверить:**
- Достаточно ли foregroundServiceType="microphone" для Android 14+?
- Не убьёт ли система приложение при долгой работе?
- Нужен ли wake lock?
- Есть ли проблемы с батареей?

### Задача 3: Backend Proxy для токена

**Требование:** Токен Telegram бота не должен быть зашит в APK.

**Варианты:**
- Сделать простой HTTP-прокси на сервере (Python/FastAPI) который принимает запросы от приложения
- Приложение шлёт аудио на сервер → сервер отправляет в Telegram
- Безопасность: токен хранится только на сервере

## 5. Варианты wake word реализации (нужен анализ)

### Вариант A: Porcupine (Picovoice) — рекомендуется
- Размер: ~2 MB к APK
- Офлайн: да
- Русский: да (встроенные ключевые слова + кастомные)
- Flutter SDK: `porcupine_flutter: ^4.0.0`
- Цена: бесплатно (AccessKey для разработки)
- Сложность интеграции: низкая (3-5 строк кода)
- Точность: высокая

### Вариант B: Vosk
- Размер: ~50 MB к APK (модель русского языка)
- Офлайн: да
- Русский: да
- Flutter SDK: нет прямого (через MethodChannel)
- Сложность: средняя (нужно писать Java/Kotlin wrapper)
- Точность: высокая (полный ASR, а не просто wake word)

### Вариант C: Серверный (через whisper)
- Приложение шлёт всё аудио на сервер
- Сервер через whisper ищет слово «Гермес»
- Если нашёл — команда принята
- Минусы: задержка, нужен интернет, трафик

### Вариант D: Android Voice Interaction API
- Стандартный Android API
- Не требует библиотек
- Минусы: не кастомизируется, системный диалог

## 6. Технические ограничения

- **Сервер:** Linux, Ubuntu/Debian 13, RAM 3.8 GB, CPU 2 cores, диск 9 GB свободно
- **Телефон:** Android (неизвестная версия, Google Play Protect блокирует самоподписанные APK)
- **Сеть:** Россия (VPN через Германию), блокировка QUIC/UDP
- **Сборка APK:** на сервере через Flutter SDK + Android SDK + Gradle (ограничение Java heap 2GB)
- **Telegram:** два бота (@GermesMCP_bot — для чата с ассистентом, @hermvois_bot — для приложения)
