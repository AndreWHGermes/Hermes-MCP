## 1. ЦЕЛЬ ПРОЕКТА
═══════════════════════════════════════════════════════════════

Android-приложение (Flutter) для голосового управления ПК через Telegram.
Пользователь (Андрей) нажимает кнопку на телефоне → говорит в микрофон →
голосовое сообщение отправляется в Telegram-бота @hermvois_bot →
серверный ассистент (Hermes) получает его, распознаёт (Whisper) → 
выполняет задачу (открыть Excel, почистить диск, найти файл) → 
отвечает голосом обратно в @hermvois_bot → приложение воспроизводит.

## 4. ПРОБЛЕМЫ (что не работает)
═══════════════════════════════════════════════════════════════

### ⛔ ПРОБЛЕМА 1: Нет записи микрофона (КРИТИЧЕСКАЯ)
Приложение не может записывать голос пользователя.
- Пакет record ^5.x несовместим с Linux-сервером сборки
  (record_linux-0.7.2 vs record_platform_interface)
- RecorderService — заглушка без нативной реализации
- wake word "Гермес" не детектируется
- Пользователь НЕ может отправлять голосовые команды

НУЖНО:
- Реализовать запись через MediaRecorder (нативный Android
  MethodChannel Java/Kotlin)
- ИЛИ найти замену record пакету, совместимую с Linux-сборкой
- ИЛИ использовать push-to-talk (запись по кнопке, без wake word)

### ⛔ ПРОБЛЕМА 2: Не работает отправка голосовых через API (КРИТИЧЕСКАЯ)
sendVoice() получает 404 Not Found при попытке отправить OGG-файл
через Telegram Bot API. При этом:
- getMe() работает (бот найден)
- sendMessage (текст) работает
- getUpdates работает
- sendDocument (файлы) — тоже 404

Гипотезы:
- Неправильный MIME тип OGG для sendVoice
- API версия Telegram блокирует с определённых IP
- Формат файла не подходит для sendVoice

### ⛔ ПРОБЛЕМА 3: getUpdates не возвращает сообщения (СРЕДНЯЯ)
После нажатия "Включить" лог показывает:
- Isolate запущен
- getUpdates уходит
- Ответ приходит с 0 обновлений

При этом:
- Пользователь пишет боту — не видно
- Другие сообщения не доходят
- В @GermesMCP_bot тоже пусто (getUpdates через меня не видит)

### ⛔ ПРОБЛЕМА 4: Foreground Service нестабилен (СРЕДНЯЯ)
- flutter_background_service v5.0.0 — API изменился
- setForegroundNotificationInfo может не быть
- При сворачивании приложения Android может убить сервис
- Требуется стабильный foreground notification + периодический keepalive

### ⛔ ПРОБЛЕМА 5: dual-bot архитектура избыточна (НИЗКАЯ)
- sendStatusToMainBot шлёт в @GermesMCP_bot, но ассистент
  не видит эти сообщения (занят своим getUpdates)
- Ассистент не может переключить режим по команде от приложения
- Можно упростить до одного бота: ассистент слушает, приложение
  пишет в тот же чат

## 6. ЧТО НУЖНО ДОБАВИТЬ (ТЗ)
═══════════════════════════════════════════════════════════════

### 6.1 Запись микрофона (P1 — обязательно)
- Реализовать через нативный MediaRecorder (MethodChannel)
- ИЛИ использовать audioplayers_record или flutter_sound
- Запись в OGG/Opus формат (Telegram поддерживает)
- Автоматическая отправка после завершения записи
- Индикатор записи (анимация, красная точка)

### 6.2 Wake word "Гермес" (P2 — желательно)
- Offline детекция через Vosk или Porcupine
- Пользователь говорит "Гермес" → приложение начинает запись
- Энергоэффективно (не держать постоянно микрофон)
- Возможность отключить wake word (push-to-talk режим)

### 6.3 Воспроизведение входящих голосовых (P1 — обязательно)
- Автоматическое воспроизведение через динамик/наушники
- Звуковой сигнал перед воспроизведением ("пим-пим")
- Кнопка "пауза/пропустить" (если длинное сообщение)

### 6.4 Foreground Service стабильный (P1 — обязательно)
- Иконка в статус-баре с подписью "Hermes Voice"
- Keepalive (периодический пинг к Telegram)
- Автоперезапуск при убийстве системой
- Низкое энергопотребление

### 6.5 Отправка голосовых через Telegram API (P1 — обязательно)
- Исправить 404 ошибку sendVoice
- Альтернатива: sendAudio или sendDocument с audio/ogg
- Проверить формат файла (OGG Opus, 48000Hz, mono)

### 6.6 UI/UX улучшения (P3 — косметика)
- Анимация пульсации при ожидании команд
- Больше звуковых эффектов (входящее сообщение, ошибка)
- Настройки: выбор микрофона, громкость, язык
- Режим "тихий" (без звуков, только вибрация)
- Тёмная тема (уже частично есть)

### 6.7 Автообновление (P2 — желательно)
- Скачивание новой версии APK из чата
- Автоматическая установка через Activity
- Уведомление о новой версии

### 6.8 Интеграция с ПК (P2 — желательно)
- Команда "открой Excel" → Open Interpreter на Windows
- Команда "почисти диск" → анализ + очистка
- Команда "найди файл" → поиск
- Результат → скриншот экрана ПК → голосовой ответ

## 7. СЕРВЕРНАЯ ЧАСТЬ (Hermes)
═══════════════════════════════════════════════════════════════

- Сервер: 91.132.57.16 (acetennis.ru)
- OS: Linux (Debian trixie)
- RAM: 3.8 GB всего, ~1 GB свободно
- LLM: DeepSeek Chat (основной), OpenRouter (запасной)
- TTS: Piper (ru_RU-irina-medium) — офлайн, качественный русский голос
- STT: faster-whisper (local, model=base)
- Сборка APK: Flutter 3.29.2 + Android SDK + OpenJDK 21 + Gradle (Xmx2g)
- Telegram: @GermesMCP_bot (где общается с Андреем)

Поток обработки голосового сообщения:
1. Андрей отправляет голосовое в @hermvois_bot
2. Hermes слушает @hermvois_bot (отдельный getUpdates)
3. Получает file_id → загружает OGG → faster-whisper → текст
4. DeepSeek анализирует запрос
5. Выполняет (Python, Open Interpreter на ПК или команды на сервере)
6. Piper TTS → голосовой ответ → @hermvois_bot
7. Приложение воспроизводит ответ
